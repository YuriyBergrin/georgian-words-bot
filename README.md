# Georgian Words Bot

Telegram-бот для изучения грузинских слов на `aiogram 3` с PostgreSQL, Redis и Alembic.

![Georgian Words Bot Logo](./description_image.png)

## Что уже реализовано

- регистрация пользователя по `/start` (без дублей по `telegram_id`)
- `/help` и кнопка `❓ Помощь` (текст из `app/help.txt`)
- роли через `ADMIN_TELEGRAM_IDS` (админские функции скрыты для обычных пользователей)
- `ADMIN_TELEGRAM_IDS` используется как bootstrap-суперадмины
- управление админами через команды (доступно bootstrap-суперадмину):
  - `/admins`
  - `/add_admin <telegram_id>`
  - `/remove_admin <telegram_id>`
- главное меню (`ReplyKeyboardMarkup`) с разными кнопками для админа и пользователя
- FSM на Redis (`RedisStorage`) для устойчивости к рестартам бота
- админ-функции словаря:
  - добавление слова через FSM
  - выбор темы из существующих или ввод новой темы
  - импорт слов пачкой из текста (`georgian | russian | topic`)
  - режим проверки импорта `dry-run`
  - лимит импорта: до 50 строк за один запуск
  - экспорт всех слов в `.txt` (`georgian | russian | topic`)
  - поиск слов с пагинацией (`🔎 Найти слово`)
  - редактирование и удаление слова из режима поиска
- тренировка слов:
  - выбор режима: все слова или конкретная тема
  - смешанное направление перевода (`GE↔RU`)
  - для части вопросов — выбор ответа из 4 вариантов
  - проверка без учета регистра и лишних пробелов
  - автоматический переход к следующему слову
  - при ошибке слово повторяется через одно слово
  - после завершения темы: сообщение о завершении и кнопка повтора темы
- интервальное повторение (SRS):
  - персональный прогресс по слову и направлению (`GE→RU` / `RU→GE`)
  - `level`, `streak_correct`, `next_review_at`, счетчики правильных/ошибок
  - мягкие интервалы на ранних уровнях и более длинные на высоких
- статистика пользователя:
  - серия дней подряд (`streak`)
  - слов в базе
  - сколько слов уже тренировал
  - сколько доступно к повторению сейчас
  - общее число правильных и ошибочных ответов
- админ-аналитика:
  - топ сложных слов по количеству ошибок

## Стек

- Python 3.13+
- aiogram 3
- SQLAlchemy (async)
- asyncpg
- Redis
- Alembic
- Loguru

## Быстрый старт

1. Создать и активировать виртуальное окружение.
2. Установить зависимости:

```bash
pip install -r requirements.txt
```

3. Заполнить `.env` (можно взять `.env.example`):

- `BOT_TOKEN`
- `DATABASE_URL`
- `REDIS_URL`

4. Поднять инфраструктуру (если используешь docker):

```bash
docker compose up -d
```

5. Применить миграции:

```bash
alembic upgrade head
```

6. Запустить бота:

```bash
python main.py
```

## Railway

- Рекомендуемый запуск через `Procfile`:

```procfile
worker: sh -c "alembic upgrade head && python main.py"
```

- Обязательные переменные окружения:
  - `BOT_TOKEN`
  - `DATABASE_URL`
  - `REDIS_URL`
  - `ADMIN_TELEGRAM_IDS` (например: `12345,67890`) — bootstrap-суперадмины

## Workflow миграций

Используем согласованный процесс:

1. Изменить модели
2. Сгенерировать миграцию:

```bash
alembic revision --autogenerate -m "..."
```

3. Проверить файл миграции вручную
4. Применить:

```bash
alembic upgrade head
```
